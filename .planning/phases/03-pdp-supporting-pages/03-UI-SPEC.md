## Spacing Scale

All values from `assets/haze-tokens.css`. Use CSS custom properties, not raw px values.

| Token | CSS Variable | Raw Value | Usage in Phase 3 |
|-------|-------------|-----------|------------------|
| xs | `--haze-space-xs` | 4px (0.4rem) | Icon-to-label gap in ingredient blocks; dot indicator gap |
| sm | `--haze-space-sm` | 8px (0.8rem) | Icon → material name → descriptor vertical gaps; summary padding |
| md | `--haze-space-md` | 16px (1.6rem) | Gallery dot indicator bottom offset; ingredient grid gap mobile; FAQ item gap; section side padding mobile |
| lg | `--haze-space-lg` | 32px (3.2rem) | Ingredient grid gap desktop; ritual description padding-x desktop; between-section spacing; About page column gap |
| xl | `--haze-space-xl` | 64px (6.4rem) | Ritual description section vertical padding; ingredients section vertical padding; About hero height contribution; FAQ page section vertical padding |

Exceptions:
- Gallery height: `100vw` on mobile (square frame) → `auto` on desktop (Dawn controls media height). Not a spacing token.
- Dot indicator dots: 8px diameter, 4px gap — align with xs token but expressed as `width: 8px; height: 8px` fixed dimensions.
- ATC button min-height: 44px (WCAG 2.5.5) — not a spacing token, applied as `min-height: 44px` directly.
- Dawn's `main-product` internal padding — overridden via targeted CSS selectors. Match `--haze-space-md` (16px) for mobile, `--haze-space-lg` (32px) for desktop side padding.
- About page hero: minimum height 60vh — not a spacing token (intentionally shorter than homepage 100vh).

Source: haze-tokens.css; CONTEXT.md D-04, D-11, D-14, D-15, D-20

## Typography

### Type Scale

Identical to Phase 2. No new sizes or weights added. All Phase 3 component text maps to exactly one of these 4 rows.

| Role | Font Family | Size | Weight | Line Height | Letter Spacing | Notes |
|------|------------|------|--------|-------------|----------------|-------|
| Display | Cormorant Garamond | 56px desktop / 36px mobile | 300 | 1.1 | 0 | Italic — used for About page hero headline only |
| Heading | Cormorant Garamond | 24px (1.5rem) | 400 | 1.2 | 0 | Product name, ritual section headline, ingredient material names, FAQ question text, About headings, FAQ page headings |
| Body | DM Sans | 16px (1rem) | 400 | 1.5 | 0 | Ritual description paragraph, kit contents richtext, ingredient descriptors, FAQ answer text, About brand copy, price display |
| Label | DM Sans | 13px (0.8125rem) | 400 | 1.3 | 0.12em | ATC button text, price label prefix, ingredient icon captions (if needed), policy page links in footer |

Weight system: **300** (Display italic only) and **400** (everything else). No third weight. Differentiate elements within a role using letter-spacing, opacity, italic style, or uppercase transform — never by adding a new weight or pixel value.

Source: Phase 2 UI-SPEC Typography (locked contract — no changes permitted)

## Color

| Role | CSS Variable | Hex Reference | Usage in Phase 3 |
|------|-------------|---------------|-----------------|
| Dominant (60%) | `--haze-color-cream` | #F7F3EE | PDP background, ritual description section, ingredients section, gallery background, FAQ page, About page background |
| Secondary (30%) | `--haze-color-blush` | #E8D5CE | Gallery dot active state, ingredient block hover tint, About page hero subtle overlay if needed |
| Accent (10%) | `--haze-color-terracotta` | #C4714A | ATC button fill, ingredient SVG icon stroke, gallery dot active state |
| Text | `--haze-color-dark` | #2C2420 | All body copy, headings, price, FAQ text, About page copy |
| Destructive | n/a | n/a | No destructive actions in Phase 3 |

Accent (`--haze-color-terracotta`) reserved for:
- ATC button background (`buy_buttons` block override)
- Ingredient icon SVG strokes (all 4 icons — same pattern as Phase 2 value prop icons)
- Gallery dot indicator active state (active dot only — inactive dots use `--haze-color-dark` at 30% opacity)
- These three elements only. Terracotta must not appear on any other element in Phase 3.

Color scheme mapping (Dawn `color-scheme-N` classes):
- `color-scheme-1` (cream background + terracotta buttons) — PDP main section, ritual description, ingredients section, FAQ page, About page
- About page hero may use `color-scheme-1` with a dark image overlay (same scrim pattern as homepage hero: `rgba(44, 36, 32, 0.25)`)

Rule: Never hardcode hex in section CSS files. Use `--haze-color-*` aliases for custom Haze sections. Dawn's `main-product` Dawn scheme colors use `rgb(var(--color-background))` / `rgb(var(--color-button))` from settings_data.json — override via targeted CSS specificity in `haze-pdp.css`.

### PDP Gallery: CSS Scroll-Snap Overlay (PDP-01)

**Target elements** (Dawn's gallery markup via `snippets/product-media-gallery.liquid`):
- `.product__media-list` — scroll container
- `.product__media-item` — individual slide

**CSS overrides in `haze-pdp.css`:**

```
/* Mobile: horizontal scroll-snap */
@media screen and (max-width: 749px) {
  .product__media-list {
    scroll-snap-type: x mandatory;
    overflow-x: scroll;
    -webkit-overflow-scrolling: touch;
    scrollbar-width: none;          /* Firefox */
  }
  .product__media-list::-webkit-scrollbar { display: none; }

  .product__media-item {
    scroll-snap-align: start;
    flex-shrink: 0;
    width: 100%;
  }
}
```

**Dot indicator (`haze-pdp-gallery.js` renders dots into DOM):**
- 4 dots (matching 4 placeholder images)
- Position: absolute, bottom `var(--haze-space-md)` (16px) from gallery bottom edge, centered horizontally
- Dot size: 8px × 8px circle
- Inactive dot: `--haze-color-dark` at 30% opacity
- Active dot: `--haze-color-terracotta` (solid, full opacity)
- Gap between dots: `var(--haze-space-xs)` (4px)
- Transition: background-color 200ms ease when active state changes
- Dot click: `scrollIntoView({ behavior: 'smooth', block: 'nearest', inline: 'start' })` to matching slide

**JS sync mechanism (IntersectionObserver):**
- Observer root: `.product__media-list` scroll container
- Threshold: 0.5 (dot becomes active when slide is ≥50% visible)
- On intersection: remove `.is-active` from all dots, add `.is-active` to matching dot by index

**Placeholder images:**
- `haze-placeholder-pdp-1.jpg` — full kit hero angle
- `haze-placeholder-pdp-2.jpg` — silk comb close-up
- `haze-placeholder-pdp-3.jpg` — brush close-up
- `haze-placeholder-pdp-4.jpg` — lifestyle / gift box
- All tagged `{%- comment -%} PLACEHOLDER {%- endcomment -%}` for grep-findability

## Accessibility Contract

- All `<img>` elements must have descriptive `alt` text — format: `"{material name} — Haze Hair Ritual Kit"` for gallery images, `"Haze brand story"` for About hero
- Gallery scroll container: `role="region" aria-label="Product images"` on `.product__media-list`
- Dot indicator: each dot gets `aria-label="Image {N} of 4"` and `aria-current="true"` on active dot
- ATC button: Dawn renders native `<button type="submit">` — do not override; accessible by default
- `<details>/<summary>` FAQ: natively accessible — keyboard-activatable, announces open/close state
- `<summary>` text must be the full question (not abbreviated) — screen readers announce the `<summary>` as the control label
- About page hero: decorative background image gets `role="img" aria-label="Haze hair care editorial photograph"` if implemented as CSS background; or standard `alt` if `<img>` tag
- Color contrast: dark `#2C2420` on cream `#F7F3EE` = ~12:1 (AAA); cream text on terracotta `#C4714A` — verify 4.5:1 minimum for Label size (13px); use `#FFFFFF` if calculation fails at 13px
- Touch targets: ATC button min-height 44px (WCAG 2.5.5); gallery dots min 44px tap area via padding even if visual dot is 8px
- Focus ring: inherit Dawn's default focus styles — do not override with Phase 3 CSS
